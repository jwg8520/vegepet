# VegePet Wireframe Notes

## 1. 문서 목적
이 문서는 VegePet의 와이어프레임 구조를 텍스트로 정리한 기준 문서다.  
Cursor AI와 개발자는 이 문서를 참고하여 Flutter UI 구조, 화면 흐름, 메뉴 구조, 화면 우선순위를 일관되게 유지해야 한다.

이 문서는 실제 와이어프레임 이미지의 해석 기준이며, 구현 시 화면 구조가 흔들리지 않도록 하기 위한 문서다.

---

## 2. 전체 화면 구조 개요

VegePet은 가로형 앱이며, 중심 화면은 마당 홈이다.  
사용자는 앱 실행 직후 게스트 상태로 바로 체험을 시작하며, 마당 화면을 중심으로 현재 펫과 상호작용하고 메뉴를 통해 다른 기능 화면으로 이동한다.

전체 구조는 크게 두 영역으로 나뉜다.

### 2.1 게임/상호작용 영역
- 마당 홈
- 펫 정보 패널
- 식단 인증 진입
- 놀아주기
- 분양권 사용
- 도감 확인

### 2.2 관리/운영 영역
- 프로필
- 식단일지
- 가방
- 상점
- 설정
- 도움말
- 스토리
- 이메일 연동

---

## 3. 앱 시작 흐름

### 3.1 기본 시작 방식
- 앱 첫 실행 시 강제 로그인 화면을 먼저 보여주지 않는다.
- 사용자는 게스트 상태로 바로 체험을 시작한다.
- 첫 진입 후 펫 분양 및 마당 진입 흐름으로 연결된다.

### 3.2 UX 원칙
- 초반 진입 장벽을 최소화한다.
- 사용자가 앱의 재미를 체험한 뒤 계정 연동 필요성을 이해하게 만든다.

---

## 4. 메인 홈 화면

### 4.1 기본 개념
마당 홈은 VegePet의 메인 허브 화면이다.  
앱 실행 후 핵심 활동은 대부분 이 화면을 중심으로 이루어진다.

### 4.2 홈 화면 구성 요소
- 가로형 마당 배경
- 집 및 배경 오브젝트
- 현재 육성 중인 펫
- 성숙기에 도달해 거주 중인 펫들
- 상단 또는 우측 상단 메뉴 버튼
- 상태/알림성 아이콘
- 펫 상호작용 진입 요소

### 4.3 홈 화면 역할
- 현재 펫 상태를 직관적으로 보여주는 역할
- 식단 인증, 놀아주기, 쓰다듬기 등 핵심 행동의 출발점
- 메뉴 진입의 중심 화면

### 4.4 홈 화면 구현 원칙
- 앱은 가로형 기준으로 구현한다
- 마당은 항상 메인 배경 역할을 한다
- 현재 육성 중인 펫이 사용자에게 가장 먼저 보여야 한다
- 성숙기 펫은 최대 6마리까지 마당에 거주할 수 있다
- Flame은 후반에 마당 이동, 애니메이션, 상호작용 구현에 활용한다

---

## 5. 펫 정보 패널

### 5.1 기본 개념
펫 정보 패널은 마당 화면에서 펫을 터치하거나 특정 UI를 통해 열리는 핵심 패널이다.  
사용자는 이 패널을 통해 펫 상태를 확인하고 주요 행동을 실행한다.

### 5.2 패널 포함 정보
- 펫 대표 이미지
- 펫 이름
- 펫 종족
- 성장 단계
- 성장 진행 상태
- 상호작용 버튼

### 5.3 주요 액션
- 식단 인증 진입
- 놀아주기
- 쓰다듬기

### 5.4 구현 원칙
- 식단 인증, 놀아주기, 쓰다듬기는 홈에서 바로 접근 가능해야 한다
- 펫 관련 핵심 액션은 가능하면 이 패널에 모은다
- 상호작용 버튼은 초보자도 직관적으로 이해할 수 있어야 한다

---

## 6. 식단 인증 화면

### 6.1 기본 개념
식단 인증은 펫 성장의 핵심 루프 중 하나이며, 반드시 실시간 카메라 기반으로 진행한다.

### 6.2 화면 구조
- 실시간 카메라 프리뷰
- 촬영 버튼
- 안내 문구
- 식단 인증 결과 확인 흐름

### 6.3 핵심 규칙
- 사진첩 업로드는 허용하지 않는다
- 실시간 카메라만 허용한다
- 하루 2회만 가능하다

### 6.4 상태 예외 처리
- 아직 배고픔 상태가 아닐 경우 진입 제한 가능
- 이미 해당 시간대 인증을 완료했으면 재촬영 제한 가능
- 결과 확인 화면에서 판정 결과와 보상 애정도를 보여줄 수 있다

### 6.5 구현 원칙
- MVP 단계에서는 우선 기능 흐름이 중요하며, 디자인은 더미 UI여도 된다
- 카메라 기능은 모바일 기준 구현을 우선 고려한다
- 웹 테스트에서는 카메라 대체 흐름이 있을 수 있으나 실제 서비스 기준은 실시간 카메라다

---

## 7. 첫 식단 성공 후 계정 연동 유도 UX

### 7.1 노출 시점
- 사용자가 첫 식단 사진을 성공적으로 등록하고 먹이주기를 완료한 직후

### 7.2 팝업 문구
- "헉! 폰을 바꾸거나 앱이 지워지면 귀여운 베지펫이 사라져요! 😢 지금 설정에서 이메일 연동을 진행할까요?"

### 7.3 버튼
- 지금 연동하기
- 나중에 할게요

### 7.4 버튼 동작
- 지금 연동하기: 설정 > 이메일 연동 메뉴로 이동
- 나중에 할게요: 팝업 닫기, 계속 플레이

### 7.5 UX 원칙
- 앱 시작 직후에는 계정 연동 유도 문구를 띄우지 않는다
- 사용자가 앱의 핵심 가치를 체감한 후에만 연동을 유도한다
- 거절해도 플레이가 막히지 않아야 한다
- 이후에도 설정에서 언제든 이메일 연동이 가능해야 한다

---

## 8. 놀아주기 화면

### 8.1 기본 개념
놀아주기는 펫과 상호작용하여 애정도를 올리는 행동이다.  
와이어프레임상 별도의 진입 화면 또는 짧은 인터랙션 화면으로 구성된다.

### 8.2 특징
- 홈에서 펫 정보 패널을 통해 진입
- 단순 버튼 처리일 수도 있으나, 추후 짧은 미니게임 형태로 확장 가능
- 하루 사용 횟수 제한이 있을 수 있다

### 8.3 구현 원칙
- MVP에서는 복잡한 미니게임보다 단순한 상호작용 흐름을 우선 구현한다
- 이후 Flame을 통해 연출 강화 가능

## 9. 메뉴 구조

### 9.1 기본 개념
홈 화면에서 메뉴 버튼을 누르면 주요 화면으로 이동 가능한 메뉴가 열린다.

### 9.2 메뉴 항목
- 프로필
- 식단일지
- 가방
- 상점
- 도감
- 스토리
- 도움말
- 설정

### 9.3 구현 원칙
- 메뉴는 초보자가 이해하기 쉽게 명확한 명칭으로 유지한다
- MVP에서는 메뉴 구조를 단순하게 유지한다
- 과도한 하위 메뉴 분기는 피한다

## 11. 식단일지 화면

### 11.1 기본 개념
식단일지는 사용자의 날짜별 식단 인증 기록과 메모를 확인하는 화면이다.

### 11.2 구조
- 월간 캘린더 화면
- 날짜별 상세 화면

### 11.3 상세 화면 포함 요소
- 식단 기록 이미지 슬롯
- 체중 기록
- 메모 또는 코멘트
- 식단 인증 결과 요약 가능

### 11.4 구현 원칙

- 하루 2회 식단 인증 구조와 자연스럽게 연결되어야 한다

## 20. Cursor AI 작업 기준

Cursor AI는 이 문서를 VegePet의 화면 구조 기준 문서로 사용해야 한다.

### 반드시 지켜야 할 해석 기준
- VegePet은 가로형 앱이다
- 앱 첫 화면은 강제 로그인 화면이 아니라 체험 흐름이어야 한다
- 메인 홈은 마당 화면이다
- 식단 인증은 홈의 펫 정보 패널에서 진입한다
- 첫 식단 성공 직후 이메일 연동 유도 팝업을 고려해야 한다
- 팝업에서 "지금 연동하기"를 누르면 설정 > 이메일 연동으로 이동해야 한다
- 설정에는 이메일 연동 메뉴와 계정 상태 표시가 있어야 한다
- 메뉴 항목은 프로필, 식단일지, 가방, 상점, 도감, 스토리, 도움말, 설정을 기준으로 한다
- 분양/도감 화면은 항상 고양이 3종 + 강아지 3종 총 6종 기준으로 해석한다
- 상점의 가구는 원목 테이블, 원목 의자, 캣휠, 텐트만 사용한다
- 장난감은 뼈다귀 인형, 실뭉치만 사용한다
- 식단 인증은 실시간 카메라만 허용하며 사진첩 업로드는 허용하지 않는다
- 성숙기 도달 펫은 최대 6마리까지 마당 거주 가능하다
- Flame은 초반 전체 앱이 아니라 마당/펫 상호작용 중심으로 후반 적용한다

## 21. 한 줄 요약
VegePet의 와이어프레임은 가로형 마당 홈을 중심으로 펫 육성, 식단 인증, 상점, 가방, 도감, 식단일지 등 핵심 기능이 연결되는 구조이며, 사용자는 게스트로 바로 체험을 시작하고 첫 식단 성공 후 이메일 연동을 유도받으며, 구현은 더미 UI → 기능 연결 → 실제 에셋 적용 → Flame 강화 순서로 진행한다.
