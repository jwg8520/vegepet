# VegePet Wireframe Notes

## 1. 문서 목적
이 문서는 VegePet 최신 와이어프레임(v2) 기준의 화면 구조와 흐름을 텍스트로 정리한 기준 문서다.  
Cursor AI와 개발자는 이 문서를 참고해 Flutter UI 구조, 화면 전환, 기능 배치, 메뉴 우선순위를 일관되게 유지해야 한다.

이 문서는 실제 와이어프레임 이미지를 해석한 기준 문서이며, 이후 더미 UI → 실제 UI → 에셋 적용 → Flame 강화 순서로 구현할 때 기준점으로 사용한다.

---

## 2. 최신 와이어프레임 핵심 구조

최신 와이어프레임은 마당 홈을 중심으로 크게 두 축으로 정리된다.

### 2.1 좌측 상단 축: 펫 정보 / 직접 상호작용
- 마당 화면에서 현재 펫을 중심으로 상태를 확인한다.
- 펫 정보 패널을 열어 현재 펫 정보와 행동 버튼에 접근한다.
- 이 축은 식단 인증, 놀아주기, 반응 확인 등 즉시 상호작용 흐름에 집중한다.

### 2.2 우측 상단 축: 게임 기능 메뉴
- 우측 상단 2x2 메뉴 버튼을 누르면 기능 허브로 진입한다.
- 이 축은 프로필, 설정, 식단일지, 상점, 가방, 도감, 스토리, 도움말 같은 관리/조회 화면에 집중한다.

즉, 최신 와이어프레임은  
**좌측 = 펫과 직접 상호작용하는 영역**  
**우측 = 기능 메뉴로 진입하는 영역**  
으로 해석한다.

## 3. 앱 시작 흐름

### 3.1 현재 개발 기준
현재 개발 단계에서는 아래 흐름을 우선 구현한다.

1. 익명 로그인
2. 프로필 입력
3. 첫 펫 분양
4. 마당 진입
5. 식단 인증 / 상호작용
6. 성장 및 다음 루프 진행

### 3.2 후순위로 나중에 추가할 것
아래 두 항목은 최신 와이어프레임 및 기획 흐름상 존재 가능하지만, 현재 뼈대 단계에서는 생략 가능하다.

- 세계관 스토리 일러스트 시작 화면
- 육성 방법 소개 팝업

### 3.3 최종 목표 흐름
정식 구현 기준 목표 흐름은 아래와 같다.

1. 로그인(내부적으로는 익명 시작)
2. 베지펫 세계관 스토리 일러스트
3. 프로필 입력
4. 분양 펫 선택
5. 육성 방법 소개
6. 마당 육성
7. 식단 인증
8. 애정도 증가
9. 성장
10. 성숙기 완료
11. 다음 펫 분양권 획득

## 4. 메인 홈 화면

### 4.1 기본 개념
마당 홈은 VegePet의 메인 허브 화면이다.  
앱 실행 후 핵심 활동은 대부분 이 화면을 중심으로 이루어진다.

### 4.2 홈 화면 구성 요소
- 가로형 마당 배경
- 집 및 배경 오브젝트
- 현재 육성 중인 펫
- 성숙기에 도달해 거주 중인 펫들
- 좌측 상단 펫 정보 진입 요소
- 우측 상단 2x2 메뉴 버튼
- 펫 상태/반응/상호작용 트리거

### 4.3 홈 화면 역할
- 현재 펫 상태를 직관적으로 보여주는 역할
- 식단 인증, 놀아주기, 반응 확인 등 핵심 행동의 출발점
- 메뉴 진입의 중심 화면

### 4.4 홈 화면 구현 원칙
- 앱은 가로형 기준으로 구현한다.
- 마당은 항상 메인 배경 역할을 한다.
- 현재 육성 중인 펫이 사용자에게 가장 먼저 보여야 한다.
- 성숙기 펫은 최대 6마리까지 마당에 거주할 수 있다.
- Flame은 후반에 마당 이동, 애니메이션, 상호작용 구현에 활용한다.

## 5. 좌측 상단: 펫 정보 패널

### 5.1 기본 개념
펫 정보 패널은 마당 화면에서 현재 펫의 상태와 즉시 행동을 확인하는 핵심 UI다.  
최신 와이어프레임 기준으로 좌측 상단 흐름의 시작점 역할을 한다.

### 5.2 패널 포함 정보
- 펫 대표 이미지 또는 아이콘
- 펫 이름 또는 종족 정보
- 성장 단계
- 상태 정보
- 행동 버튼

### 5.3 주요 행동
최신 와이어프레임 기준으로 펫 정보 패널에서는 아래 행동 흐름으로 연결된다.

- 식단 인증 진입
- 놀아주기 진입
- 펫 반응/상태 메시지 확인
- 특정 조건에서 행동 불가 안내 확인

### 5.4 구현 원칙
- 펫 직접 상호작용은 이 패널에서 시작되게 한다.
- 사용자가 펫과 “지금 바로 할 수 있는 행동”을 직관적으로 이해할 수 있어야 한다.
- 홈 화면의 다른 기능 메뉴와 역할이 섞이지 않도록 한다.

## 6. 식단 인증 화면

### 6.1 기본 개념
식단 인증은 펫 성장의 핵심 루프 중 하나이며, 반드시 실시간 카메라 기반으로 진행한다.

### 6.2 최신 와이어프레임 해석
펫 정보 패널에서 식단 인증을 누르면 식단 촬영 화면으로 이동한다.  
이후 촬영 성공/실패/행동 제한 상태에 따라 별도 메시지 또는 안내 화면이 이어질 수 있다.

### 6.3 화면 구조
- 실시간 카메라 프리뷰
- 촬영 버튼
- 안내 문구
- 식단 인증 결과 확인 흐름

### 6.4 핵심 규칙
- 사진첩 업로드는 허용하지 않는다.
- 실시간 카메라만 허용한다.
- 하루 2회만 가능하다.

### 6.5 상태 예외 처리
최신 와이어프레임 기준으로 아래와 같은 예외 메시지/안내 상태가 존재할 수 있다.

- 아직 식단 인증 가능한 상태가 아닐 때
- 이미 해당 시간대 인증을 완료했을 때
- 식단 인증 결과에 따른 짧은 메시지 노출

### 6.6 구현 원칙
- MVP 단계에서는 우선 기능 흐름이 중요하며, 디자인은 더미 UI여도 된다.
- 카메라 기능은 모바일 기준 구현을 우선 고려한다.
- 웹 테스트에서는 대체 흐름이 있을 수 있으나 실제 서비스 기준은 실시간 카메라다.

## 7. 펫 반응 / 상태 메시지 흐름

### 7.1 기본 개념
최신 와이어프레임은 펫 행동 결과에 따라 짧은 메시지 박스 또는 상태 안내를 보여주는 흐름이 포함되어 있다.

### 7.2 포함 가능한 상태
- 식단 인증 가능/불가 상태
- 펫 반응 메시지
- 상호작용 후 짧은 피드백
- 특정 조건 충족 전 행동 제한 안내

### 7.3 구현 원칙
- 메시지는 짧고 즉각적인 반응 형태로 제공한다.
- 장문의 설명보다는 “펫의 현재 상태를 알려주는 감정형 메시지”에 가깝게 구현한다.
- 팝업, 토스트, 말풍선형 카드 중 하나로 구현 가능하다.

## 8. 첫 식단 성공 후 이메일 연동 유도 UX

### 8.1 노출 시점
- 사용자가 첫 식단 사진을 성공적으로 등록하고 먹이주기를 완료한 직후

### 8.2 팝업 문구
- "헉! 폰을 바꾸거나 앱이 지워지면 귀여운 베지펫이 사라져요! 😢 지금 설정에서 이메일 연동을 진행할까요?"

### 8.3 버튼
- 지금 연동하기
- 나중에 할게요

### 8.4 버튼 동작
- 지금 연동하기: 설정 > 이메일 연동 메뉴로 이동
- 나중에 할게요: 팝업 닫기, 계속 플레이

### 8.5 UX 원칙
- 앱 시작 직후에는 계정 연동 유도 문구를 띄우지 않는다.
- 사용자가 앱의 핵심 가치를 체감한 후에만 연동을 유도한다.
- 거절해도 플레이가 막히지 않아야 한다.
- 이후에도 설정에서 언제든 이메일 연동이 가능해야 한다.

---

## 9. 우측 상단: 게임 기능 메뉴

### 9.1 기본 개념
홈 화면 우측 상단의 2x2 메뉴 버튼은 VegePet의 기능 허브 진입점이다.

### 9.2 역할
- 사용자가 메인 상호작용 이외의 화면으로 이동하도록 돕는다.
- 프로필, 설정, 식단일지, 상점, 가방, 도감, 스토리, 도움말 같은 기능을 한 곳에서 진입하게 한다.

### 9.3 구현 원칙
- 메뉴는 초보자가 이해하기 쉽게 명확한 명칭으로 유지한다.
- MVP에서는 메뉴 구조를 단순하게 유지한다.
- 과도한 하위 메뉴 분기는 피한다.

## 10. 프로필 화면

### 10.1 기본 개념
프로필 화면은 사용자의 기본 정보와 목표를 보여주는 화면이다.

### 10.2 포함 정보
- 닉네임
- 성별
- 식단목적
- 다짐 한마디

### 10.3 역할
- 앱 개인화 정보 조회 및 수정
- 식단 AI 판단에 필요한 사용자 기본 정보 제공

### 10.4 구현 원칙
- 현재 기획 기준으로 프로필 입력은 초반 필수 단계다.
- 프로필 정보는 첫 식단 인증 전에 반드시 저장되어 있어야 한다.

앱 설정, 계정 관리, 고객지원성 메뉴를 제공한다.

- 게스트 상태 사용자는 연동 필요성을 이해할 수 있어야 한다.
- 연동 완료 사용자는 현재 상태를 확인할 수 있어야 한다.

## 12. 식단일지 화면

### 12.1 기본 개념
식단일지는 사용자의 날짜별 식단 인증 기록과 메모를 확인하는 화면이다.

### 12.2 구조
- 월간 캘린더 화면
- 날짜별 상세 화면

### 12.3 상세 화면 포함 요소
- 식단 기록 이미지 슬롯
- 체중 기록
- 메모 또는 코멘트
- 식단 인증 결과 요약

### 12.4 구현 원칙
- 캘린더와 상세 화면을 분리한다.
- 하루 2회 식단 인증 구조와 자연스럽게 연결되어야 한다.

### 14.1 기본 개념

### 14.2 카테고리

## 15. 도감 화면

### 15.1 기본 개념
도감은 사용자가 육성 완료한 펫의 수집 현황을 보여주는 화면이다.

### 15.2 구조
- 고양이 섹션
- 강아지 섹션

### 15.3 역할
- 수집 진행률 확인

- 종족별 정리

### 15.4 구현 원칙

- 도감 UI는 이 기준을 벗어나지 않도록 한다.

## 18. 구현 우선순위 기준

### 18.1 1차 구현
- 익명 시작
- 프로필 입력
- 첫 펫 분양
- 마당 화면
- 펫 정보 패널 더미 UI

## 19. Cursor AI 작업 기준

Cursor AI는 이 문서를 VegePet의 최신 화면 구조 기준 문서로 사용해야 한다.

### 반드시 지켜야 할 해석 기준
- VegePet은 가로형 앱이다.
- 메인 홈은 마당 화면이다.
- 좌측 상단은 펫 정보 및 직접 상호작용 축이다.
- 우측 상단은 게임 기능 메뉴 축이다.
- 앱 시작 직후에는 강제 로그인 화면을 먼저 보여주지 않는다.
- 현재 구현 기준으로는 익명 시작 → 프로필 입력 → 펫 분양 → 마당 진입 흐름을 우선한다.
- 식단 인증은 펫 정보 축에서 진입한다.
- 첫 식단 성공 직후 이메일 연동 유도 팝업을 고려해야 한다.
- 팝업에서 "지금 연동하기"를 누르면 설정 > 이메일 연동으로 이동해야 한다.
- 설정에는 이메일 연동 메뉴와 계정 상태 표시가 있어야 한다.
- 분양/도감 화면은 항상 고양이 3종 + 강아지 3종 총 6종 기준으로 해석한다.
- 상점의 가구는 원목 테이블, 원목 의자, 캣휠, 텐트만 사용한다.
- 장난감은 뼈다귀 인형, 실뭉치만 사용한다.
- 식단 인증은 실시간 카메라만 허용하며 사진첩 업로드는 허용하지 않는다.
- 성숙기 도달 펫은 최대 6마리까지 마당 거주 가능하다.
- 스토리 일러스트와 육성 방법 소개는 현재 뼈대 단계에서는 생략 가능하다.
- Flame은 초반 전체 앱이 아니라 마당/펫 상호작용 중심으로 후반 적용한다.

## 20. 한 줄 요약
VegePet 최신 와이어프레임은 가로형 마당 홈을 중심으로 좌측 상단의 펫 직접 상호작용 축과 우측 상단의 기능 메뉴 축으로 구성되며, 현재 구현은 익명 시작 → 프로필 입력 → 펫 분양 → 마당 → 식단 인증 순서로 진행하고, 이후 스토리/실제 에셋/Flame 강화가 추가된다.
