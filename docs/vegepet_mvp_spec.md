# VegePet MVP Spec

## 1. 프로젝트 개요
VegePet은 식단관리와 펫 육성을 결합한 가로형 모바일 앱이다.  
사용자는 별도 회원가입 없이 바로 앱을 체험할 수 있고, 펫 분양, 식단 인증, 상호작용을 통해 펫을 성장시킨다.  
사용자가 앱의 가치를 체감한 뒤 원할 때 이메일 연동을 진행하는 구조를 채택한다.

- 앱 형태: 가로형 앱
- 기술 스택: Flutter + Supabase
- MVP 목표: 식단 인증과 펫 육성의 핵심 루프를 검증하는 것
- 계정 전략: 게스트 체험 우선, 이후 이메일 연동

## 3. MVP 핵심 플레이 흐름

1. 사용자는 앱 실행 후 별도 로그인 없이 바로 게스트로 시작한다.
2. 사용자는 첫 펫 1마리를 분양받고 마당에 진입한다.
3. 사용자는 마당 화면에서 현재 펫을 확인하고 상호작용할 수 있다.
4. 사용자는 하루 2회 실시간 식단 인증을 수행할 수 있다.
5. 식단 결과에 따라 애정도가 증가한다.
6. 사용자는 놀아주기 / 쓰다듬기를 통해 추가 애정도를 획득할 수 있다.
7. 사용자는 첫 식단 사진 등록 및 먹이주기 성공 후, 이메일 연동 유도 팝업을 본다.
8. 사용자가 원하면 설정의 이메일 연동 메뉴로 이동해 계정을 연동한다.
9. 사용자는 펫이 성장 단계를 거쳐 성숙기에 도달할 때까지 육성한다.
10. 육성 완료 후 다음 펫 1마리는 랜덤 분양권으로 획득한다.
11. 새로 획득한 펫도 동일한 방식으로 육성한다.
12. 성숙기에 도달한 펫은 마당 거주 펫으로 편입된다.

## 4. 핵심 MVP 기능

### 4.1 계정 / 인증
- 앱 첫 실행 시 게스트(익명) 체험 시작
- 사용자가 원할 때 이메일 연동
- 이메일 연동 방식은 OTP 기반으로 진행

### 4.2 펫 육성
- 첫 펫 1마리 선택 후 마당에서 육성
- 육성 완료(성숙기 달성) 후 다음 펫 1마리는 랜덤 분양권으로 획득
- 동일한 육성 루프 반복

### 4.3 식단 인증
- 하루 2회 식단 인증
- 실시간 카메라 촬영 기반 인증
- 식단 결과에 따라 애정도 증가

### 4.4 상호작용
- 놀아주기
- 쓰다듬기
- 수행 결과에 따라 애정도 증가

### 4.5 관리 / 수집
- 도감
- 상점
- 가방
- 식단일지
- 설정 내 이메일 연동 메뉴

## 5. 핵심 규칙

### 5.1 식단 인증 규칙
- 식단 인증은 실시간 카메라만 허용한다.
- 사진첩 업로드는 허용하지 않는다.
- 하루 2회만 가능하다.

### 5.2 펫 배치 규칙
- 성숙기에 도달한 펫은 마당에 거주할 수 있다.
- 성숙기 도달 펫은 최대 6마리까지 마당 거주 가능하다.
- 대표 펫 개념은 최대 6마리 기준으로 운영한다.

### 5.3 성장 단계
펫의 성장 단계는 아래 순서를 따른다.

- 유아기
- 유년기
- 성장기
- 성숙기

### 5.4 계정 연동 규칙
- 앱 시작 시 계정 생성 강요를 하지 않는다.
- 사용자는 게스트 상태로 체험을 시작한다.
- 첫 식단 사진 등록 성공 후 이메일 연동 유도 팝업을 노출한다.
- 사용자가 거절해도 계속 플레이할 수 있어야 한다.
- 이메일 연동 메뉴는 설정에서 언제든 다시 접근할 수 있어야 한다.

## 6. 계정 전략

### 6.1 기본 전략
- 첫 진입은 게스트 체험이 기본이다.
- 사용자가 앱의 가치를 체험한 뒤 이메일 연동을 진행하도록 유도한다.

### 6.2 이메일 연동 유도 시점
- 첫 식단 사진 등록 및 먹이주기 성공 직후

### 6.3 이메일 연동 유도 문구
- "헉! 폰을 바꾸거나 앱이 지워지면 귀여운 베지펫이 사라져요! 😢 지금 설정에서 이메일 연동을 진행할까요?"

### 6.4 팝업 버튼
- 지금 연동하기
- 나중에 할게요

### 6.5 버튼 동작
- 지금 연동하기: 설정 > 이메일 연동 메뉴로 이동
- 나중에 할게요: 팝업 닫기, 계속 플레이 가능

### 6.6 추가 UX 원칙
- 설정 화면에 이메일 연동 메뉴를 명확히 둔다.
- 게스트 상태임을 설정에서 확인할 수 있어야 한다.
- 사용자가 나중에라도 이메일 연동 위치를 쉽게 찾을 수 있어야 한다.

- name
- gender

- account_type (guest / registered)

사용자 보유 아이템
- user_id
- 아이템 종류

## 11. 개발 원칙

1. 먼저 백엔드 데이터 골격을 만든다.
2. 그 다음 Flutter에서 더미 UI로 기능 흐름을 먼저 붙인다.
3. 기능 연결이 확인되면 와이어프레임 기준으로 UI를 정리한다.
4. 이후 실제 디자인 리소스를 제작하고 적용한다.
5. Flame은 마당, 펫 이동, 상호작용 등 핵심 게임 영역에 후반 적용한다.
6. 첫 진입에서는 가입 장벽을 만들지 않는다.
7. 계정 연동 유도는 사용자가 가치를 체감한 뒤에만 노출한다.

## 13. Cursor AI 작업 기준

Cursor AI는 이 문서를 VegePet 프로젝트의 기준 문서로 사용해야 한다.

### 반드시 지켜야 할 기준
- Flutter + Supabase 기반으로 제안할 것
- MVP 범위를 벗어나는 과도한 기능을 먼저 제안하지 말 것
- 앱 시작 시 강제 로그인 화면을 첫 화면으로 두지 말 것
- 기본 시작 방식은 게스트(익명) 체험으로 처리할 것
- 사용자가 원할 때 이메일 OTP 연동을 진행하는 구조로 제안할 것
- 펫 종 수는 항상 고양이 3종 + 강아지 3종 총 6종으로 취급할 것
- 가구는 원목 테이블, 원목 의자, 캣휠, 텐트만 우선 사용
- 장난감은 뼈다귀 인형, 실뭉치만 우선 사용
- 식단 인증은 실시간 카메라만 허용
- 사진첩 업로드는 허용하지 않음
- 성숙기 펫은 최대 6마리까지 마당 거주 가능
- 성장 단계는 유아기 → 유년기 → 성장기 → 성숙기 순서로 고정
- 첫 식단 성공 직후 계정 연동 유도 팝업을 고려할 것
- 이메일 연동 메뉴는 설정에서 접근 가능해야 한다

## 14. 한 줄 요약
VegePet MVP는 실시간 식단 인증과 펫 육성을 결합한 가로형 Flutter + Supabase 앱이며, 사용자는 게스트로 바로 체험을 시작하고 첫 식단 성공 후 이메일 연동을 유도받으며, 6종의 펫을 순차 육성하고 성숙기 달성 후 랜덤 분양권으로 다음 펫을 획득하는 구조를 핵심 루프로 한다.
